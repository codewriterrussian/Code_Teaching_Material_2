# Lesson 2 延伸：變數命名

---

## 本課重點

變數名稱不是隨便取的。

好的命名可以讓自己和別人更快看懂程式，也能減少錯誤。

---

## 課前暖身

請想一想：

如果手機相簿裡的照片都叫 `image1`、`image2`、`image3`，你能很快知道哪一張是「生日蛋糕」嗎？

- 如果檔名叫 `birthday_cake`，會更容易看懂。
- 程式中的變數名稱也一樣，要讓人一看就知道用途。

---

---

## 核心概念：為什麼變數要好好命名？

變數就像箱子。

箱子外面的標籤如果寫得清楚，我們就知道裡面放什麼。

變數名稱也是一種標籤。

不好的變數名稱雖然有時候程式可以執行，但人會很難讀懂，尤其程式變長之後更容易搞混。

---

## 規則一：使用有意義的命名

合理的命名會清楚說明變數用途。

例如 `user_input` 表示使用者輸入，`total_price` 表示總價格。

```python
# 合理的命名
user_input = input()
total_price = 21564

# 不合理的命名
x = 5343
y = 132135
```

生活化理解：

飲料店點餐機如果用 `x` 表示杯數、`y` 表示總價，過一段時間可能連自己都忘記。

如果寫成 `cup_count` 和 `total_price`，就比較清楚。

| 不清楚的名稱 | 比較清楚的名稱 | 可能代表的意思 |
| --- | --- | --- |
| `x` | `score` | 分數 |
| `y` | `total_price` | 總價格 |
| `a` | `user_name` | 使用者名字 |
| `b` | `is_open` | 是否開啟 |

---

## 規則二：命名格式

當變數名稱需要多個英文單字時，可以用固定格式讓它更好讀。

常見寫法如下：

```python
userInput = input()    # 小駝峰式 camelCase
user_input = input()   # 底線式 snake_case
user_Input = input()   # 不建議：大小寫混用不一致
```

Python 課程中通常建議使用 `snake_case`，也就是用小寫英文加底線。

例如：

```python
total_price
user_name
```

小提醒：

Python 會區分大小寫。

`user_input` 和 `user_Input` 對 Python 來說是不同變數。

同一份程式中建議選一種命名規則，並一直使用同一種。

---

## 規則三：避開保留字

保留字是 Python 已經拿去使用的特殊字，不能再拿來當變數名稱。

| 常見保留字 | 用途提醒 | 不要這樣命名 |
| --- | --- | --- |
| `print` | 輸出用的函式 | `print = 10` |
| `input` | 輸入用的函式 | `input = "hi"` |
| `if / else` | 條件判斷 | `if = 3` |
| `while / for` | 迴圈 | `for = 5` |
| `import / as / in` | 引入模組或語法的一部分 | `import = 1` |
| `try / except` | 錯誤處理 | `except = 2` |
| `range` | 常和迴圈搭配 | `range = 10` |

```python
# 不建議或可能造成錯誤
print = 10
input = "hello"
for = 3
```

其中 `for = 3` 會直接造成語法錯誤。

`print = 10` 雖然可能暫時被 Python 接受，但會讓原本的 `print()` 不能正常使用，所以也要避免。

---

## Section VIII. 規則四：保持同樣的命名規則

同一份程式中，命名方式要一致。

如果一開始使用 `user_name`，後面就不要突然改成 `userName` 或 `UserName`。

```python
# 較一致
user_name = "Amy"
user_score = 95
user_level = 3

# 較不一致，不建議
userName = "Amy"
user_score = 95
UserLevel = 3
```

生活化理解：

考卷填空題如果一下叫「姓名」，一下叫「名字」，一下叫 `Name`，學生會容易混亂。

程式也一樣，名稱一致可以讓閱讀速度變快。

---

## 常見錯誤

| 錯誤情況 | 原因 | 修正方式 |
| --- | --- | --- |
| 使用沒有意義的名稱，例如 `x, y, z` | 程式一長就不知道用途。 | 改成 `total_price`、`user_input` 等清楚名稱。 |
| 大小寫前後不一致 | Python 會區分大小寫。 | 固定使用一種名稱，例如 `user_name`。 |
| 使用保留字當變數名稱 | 會造成語法錯誤或覆蓋內建功能。 | 改成其他名稱，例如 `user_input`。 |
| 名稱中間有空白 | Python 變數名稱不能有空白。 | 用底線連接，例如 `total_price`。 |
| 變數名稱用數字開頭 | Python 不允許變數以數字開頭。 | 改成 `score1`，不要寫 `1score`。 |

---

## 重點複習

| 觀念 | 說明 |
| --- | --- |
| 清楚標籤 | 變數名稱要像清楚的標籤，讓人知道資料用途。 |
| 有意義的英文單字 | 建議使用 `user_name`、`total_price` 這類名稱。 |
| `snake_case` | 多個單字可使用底線連接，例如 `user_input`。 |
| 避開保留字 | 不要用 Python 保留字當變數名稱。 |
| 命名一致 | 同一份程式要保持命名規則一致。 |
